Getting Started

This guide will help you understand how to use the API configuration screen to create or edit APIs. We'll explain how to format your requests, use dynamic placeholders, and set up response rules correctly.

2

3

Define how to process the API response with keys and conditional rules.

Basic Information

Description

Provide a clear, descriptive name for your API that helps identify its purpose. This will help other users understand what this API does.

API Type

Select the appropriate type for your API:

  • Regular API: For general API endpoints used for transactions
  • Balance Check: Specifically for checking account balances
  • Offer Check: For retrieving or validating offers
  • Message: For sending SMS or notifications

Status

Set whether this API is active or inactive. Inactive APIs won't be called by the system.

Request Configuration

Understanding URL Components

Base URL: The main domain address of the API service (e.g., https://api.example.com)

Endpoint: The specific path on the server that handles this particular function (e.g., /v1/checkBalance)

Port: A specific communication channel used by the server (e.g., 8080). Default ports (80 for HTTP, 443 for HTTPS) can be left empty.

Complete URL Example: https://api.example.com:8080/v1/checkBalance

Request Port

The port number for the API endpoint (if required). Leave empty if using standard ports (80 for HTTP, 443 for HTTPS).

Request URL Endpoint

The complete URL path to the API endpoint. Include the full URL including the protocol (http:// or https://).

Example: https://api.example.com/v1/checkBalance

Request Type

Select the HTTP method for this API call:

  • GET: Retrieve data from the server
  • POST: Send data to the server
  • PUT: Update existing data on the server
  • DELETE: Remove specific data from the server

Headers

This field accepts a JSON object containing all request headers. Each header key-value pair should be formatted as a property in the JSON object.

Example: 
{
  "Content-Type": "application/json",
  "Authorization": "Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==",
  "X-API-Key": "12345678"
}

Query Parameters

These parameters will be appended to the URL as query string. Format as a JSON object.

Example: 
{
  "mobile": "[number]",
  "opcode": "[opcode]",
  "apikey": "12345678"
}

Body Parameters

These parameters will be included in the request body. Format as a JSON object.

Example: 
{
  "accountNumber": "[account]",
  "amount": "[amount]",
  "transactionType": "recharge"
}

Dynamic Placeholders

Use placeholders in your header, query parameters, and body to insert dynamic values at runtime. Placeholders are surrounded by square brackets.

Placeholder Description Example Usage
[number] Mobile or phone number "mobile": "[number]"
[account] Account number "accountNumber": "[account]"
[amount] Transaction amount "amount": "[amount]"
[opcode] Operator code "operatorCode": "[opcode]"
[message] Message text for SMS "text": "[message]"
Important: Ensure placeholder names are spelled correctly and enclosed in square brackets. The system will automatically replace these with actual values at runtime.

Response Configuration

Response Format Type

Select the expected format of the API response:

  • JSON: For structured JSON responses
  • XML: For XML formatted responses
  • Text: For plain text responses

Text Separator

Only applicable for text responses. Specify the character that separates values in the response (comma, pipe, etc.).

Response Keys

Define the keys from the API response that you want to extract and use for evaluation in rules.

Tip: For JSON responses, use dot notation for nested objects (e.g., response.status).

Response Rules

Set up conditional rules to determine how the system should handle different response scenarios.

Creating a Rule:

  1. Define Condition 1 (required): Select a response key, condition operator, and the value to compare against.
  2. Define Condition 2 (optional): Add a second condition if needed.
  3. Select an Action: What should happen when these conditions are met.
  4. Add a Forward Message: The message to send to the user when this rule is triggered.
Example Rule:
  • Condition 1: If status equals SUCCESS
  • Condition 2: If balance greater than 0
  • Action: SUCCESS
  • Forward Message: Transaction successful. Current balance: {balance}

Filters

Filters allow you to extract specific values from the API response. Use JSONPath or XPath expressions depending on your response format.

Available Filters

RF
Response Filter

Extracts the path to navigate inside the response structure to find specific values.

Example: response.data.resultCode

CF
Customer Filter

Extracts customer information from the response.

Example: response.data.customerInfo

AF
Amount Filter

Extracts transaction amount values from the response.

Example: response.data.transactionAmount

BF
Balance Filter

Extracts account balance values from the response.

Example: response.data.accountBalance

TF
Transaction ID Filter

Extracts transaction identifiers from the response.

Example: response.data.txnId

RF
Request ID Filter

Extracts request identifiers from the response.

Example: response.data.requestId

MF
Message Filter

Extracts message text from the response.

Example: response.data.message

Tip for JSON responses: Use dot notation or JSONPath syntax: 
response.data.balance
$.response.data.balance
Tip for XML responses: Use XPath syntax: 
//response/data/balance
