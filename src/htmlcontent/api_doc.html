Modern Tech Creations API Documentation

Welcome to the Modern Tech Creations API documentation. This API allows you to perform mobile recharges, check balance, and verify transaction status programmatically.

The API is designed to be simple and easy to integrate with your applications. It follows RESTful principles and returns responses in JSON format.

Authentication

All API requests require authentication using an API key. You can provide your API key in one of the following ways:

  • As a query parameter: ?key=YOUR_API_KEY
  • In the request body: {"key": "YOUR_API_KEY"}
  • As a header: X-API-Key: YOUR_API_KEY
  • As a cookie: key=YOUR_API_KEY

Example Request with API Key

GET /api/v1/mtc/checkBalance?key=YOUR_API_KEY HTTP/1.1
Host: www.moderntechcreations.com
Contact our support team to get your API key.

IP Whitelisting

For security reasons, we implement IP whitelisting. Only requests from whitelisted IP addresses will be accepted.

To add your IP address to the whitelist, please use our app or website

  • Your API key
  • IP address(es) to whitelist
  • Purpose of integration
If your IP address changes, you'll need to update your IPs.

Base URL

All API requests should be made to the following base URL:

https://www.moderntechcreations.com/api/v1/mtc

Endpoints

Recharge

GET /recharge

This endpoint is used to initiate a recharge for a mobile number or other service.

Request Parameters

Parameter Type Required Description
key string Required Your API key
opcode string Required Operator code for the service provider
number string Required Mobile number or service number
amount decimal Required Recharge amount
account string Optional Account number (if different from number)

Example Request

GET /api/v1/mtc/recharge?key=YOUR_API_KEY&opcode=AIRTEL&number=9876543210&amount=100 HTTP/1.1
Host: www.moderntechcreations.com

Success Response (200 OK)

{
  "status": "success",
  "message": "Recharge successful",
  "amount": 100,
  "number": "9876543210",
  "account": "9876543210",
  "txnId": "TXN-123-456-1620000000000"
}

Pending Response (200 OK)

{
  "status": "pending",
  "message": "Recharge is pending",
  "amount": 100,
  "number": "9876543210",
  "account": "9876543210",
  "txnId": "TXN-123-456-1620000000000"
}

Error Response (400 Bad Request)

{
  "status": "failed",
  "message": "Insufficient balance"
}
If the recharge status is "pending", use the Status Check endpoint to get the latest status.

Check Balance

GET /checkBalance

This endpoint allows you to check your current account balance.

Request Parameters

Parameter Type Required Description
key string Required Your API key

Example Request

GET /api/v1/mtc/checkBalance?key=YOUR_API_KEY HTTP/1.1
Host: www.moderntechcreations.com

Success Response (200 OK)

{
  "balance": 5000.75
}

Error Response (400 Bad Request)

{
  "status": "failed",
  "message": "User not found"
}

Status Check

GET /statuscheck

This endpoint allows you to check the status of a recharge transaction.

Request Parameters

Parameter Type Required Description
key string Required Your API key
txnid string Required Transaction ID received from the recharge endpoint

Example Request

GET /api/v1/mtc/statuscheck?key=YOUR_API_KEY&txnid=TXN-123-456-1620000000000 HTTP/1.1
Host: www.moderntechcreations.com

Success Response (200 OK)

{
  "status": "success",
  "message": "Recharge successful",
  "amount": 100,
  "number": "9876543210",
  "account": "9876543210",
  "txnId": "TXN-123-456-1620000000000"
}

Error Response (400 Bad Request)

{
  "status": "failed",
  "message": "Recharge not found"
}

Callbacks

Our API supports callbacks to notify you about the status of recharge transactions. This is particularly useful for transactions that are initially in a "pending" state.

Setting Up Callbacks

To receive callbacks, you need to set up a callback URL in your account. When a recharge transaction status changes, our system will send a GET request to your callback URL with the following parameters:

Parameter Type Description
status string Status of the transaction (success, failed, pending)
message string Detailed message about the transaction
amount decimal Recharge amount
number string Mobile number or service number
account string Account number
txnId string Transaction ID

Example Callback Request

GET /your-callback-endpoint?status=success&message=Recharge%20successful&amount=100&number=9876543210&account=9876543210&txnId=TXN-123-456-1620000000000 HTTP/1.1
Host: your-domain.com
To set up a callback URL, please contact our support team.

Error Codes

Here are the common error codes you might encounter while using our API:

Status Code Message Description
400 Keyword not found The specified operator code (opcode) is invalid or not supported
400 Insufficient balance Your account does not have enough balance for the transaction
400 Customer number is too short/long The provided number does not meet the length requirements
400 Amount is too low/high The specified amount is outside the allowed range
400 No lines available for this recharge No service available for the specified combination
400 Recharge API not available The service is temporarily unavailable
401 API key is missing No API key was provided with the request
401 Invalid API key The provided API key is invalid
401 Account is inactive Your account has been deactivated
403 Unauthorized IP address Request is coming from a non-whitelisted IP address

Integration Guide

Follow these steps to integrate our API into your application:

  1. Get API Credentials

    Contact our support team to get your API key and whitelist your IP addresses.

  2. Test Your Integration

    Use small amounts to test your integration with our API.

  3. Set Up Error Handling

    Implement proper error handling for different status codes and error messages.

  4. Handle Callbacks (Optional)

    If you want to receive real-time updates about transaction status, set up a callback URL.

    const response = await axios.get('https://www.moderntechcreations.com/api/v1/mtc/recharge', {

        key: 'YOUR_API_KEY',
        opcode: 'AIRTEL',
        number: '9876543210',

      }
    });

    console.log(response.data);
    return response.data;
  } catch (error) {
    console.error('Error:', error.response ? error.response.data : error.message);
    throw error;
  }
}
